CKA — Certified Kubernetes Administrator

📋 Exam Overview
  • Duration: 2 hours — fully hands-on, performance-based exam (no MCQs)
  • Platform: killer.sh simulator (2 free attempts with exam voucher)
  • Passing score: 66% — 17 tasks across ~25 questions
  • Open-book: kubernetes.io/docs allowed — practice navigating fast
  • Cost: $395 USD (includes 1 free retake)
  • Valid: 3 years from date of passing
🏗️ Core Architecture (25%)
  • Control plane: kube-apiserver, etcd, kube-scheduler, kube-controller-manager
  • Worker node: kubelet, kube-proxy, container runtime (containerd)
  • kubeadm init — bootstrap a cluster from scratch
  • kubeadm join — add worker nodes with token
  • etcd backup: ETCDCTL_API=3 etcdctl snapshot save — must know by heart
  • etcd restore from snapshot — restore cluster state
  • Upgrade cluster: kubeadm upgrade plan → apply → kubelet → kubectl
  • Static pods: /etc/kubernetes/manifests/ — apiserver, scheduler, etcd live here
⚙️ Workloads & Scheduling (15%)
  • Pod, ReplicaSet, Deployment, DaemonSet, StatefulSet, Job, CronJob
  • kubectl rollout status/history/undo deployment/myapp
  • Rolling update strategy: maxSurge, maxUnavailable
  • Resource requests and limits: cpu, memory — LimitRange, ResourceQuota
  • nodeSelector, nodeAffinity (required / preferred during scheduling)
  • Taints: kubectl taint nodes node1 key=val:NoSchedule
  • Tolerations in pod spec to allow scheduling on tainted nodes
  • HPA: kubectl autoscale deployment — CPU-based scaling
  • PriorityClass — eviction order for pods under resource pressure
🌐 Services & Networking (20%)
  • Service types: ClusterIP, NodePort, LoadBalancer, ExternalName
  • Ingress: rules, paths, TLS — nginx ingress controller
  • CoreDNS — service discovery: my-svc.my-ns.svc.cluster.local
  • NetworkPolicy: podSelector, namespaceSelector, ingress/egress rules
  • CNI plugins: Calico (NetworkPolicy support), Flannel, Weave
  • kubectl port-forward pod/mypod 8080:80 — local testing
  • Endpoints, EndpointSlices — how services route to pods
💾 Storage (10%)
  • PersistentVolume (PV), PersistentVolumeClaim (PVC), StorageClass
  • Access modes: ReadWriteOnce, ReadOnlyMany, ReadWriteMany
  • Reclaim policies: Retain, Recycle, Delete
  • Dynamic provisioning: StorageClass + provisioner (AWS EBS, NFS)
  • Volume types: emptyDir, hostPath, configMap, secret, nfs
  • VolumeMount vs Volume — know the YAML by heart
🔐 Security (12%)
  • RBAC: Role, ClusterRole, RoleBinding, ClusterRoleBinding
  • ServiceAccount — identity for pods to access API server
  • kubectl create role / rolebinding — imperative commands (faster in exam)
  • Secrets: Opaque, kubernetes.io/tls, kubernetes.io/dockerconfigjson
  • ConfigMap vs Secret — when to use which
  • SecurityContext: runAsUser, runAsNonRoot, readOnlyRootFilesystem
  • kubeconfig: clusters, contexts, users — kubectl config use-context
🔧 Troubleshooting (30%)
  • kubectl describe pod/node — Events section is your best friend
  • kubectl logs mypod --previous — logs from crashed container
  • CrashLoopBackOff: check logs → OOM? wrong command? missing config?
  • Pending pod: insufficient resources? node taint? no matching node?
  • ImagePullBackOff: wrong image name? private registry? no pull secret?
  • Node NotReady: kubectl describe node → check kubelet: systemctl status kubelet
  • journalctl -u kubelet -f — live kubelet logs on node
  • kubectl get events --sort-by='.lastTimestamp' — chronological events
  • kubectl exec -it pod -- /bin/sh — shell into container
  • Fixing broken kubeconfig, missing certificates, misconfigured scheduler
⚡ Exam Speed Tips
  • alias k=kubectl && export do="--dry-run=client -o yaml" — add to ~/.bashrc immediately
  • k run pod1 --image=nginx $do > pod.yaml — generate YAML fast
  • k create deploy d1 --image=nginx --replicas=3 $do — deployment YAML
  • k expose pod mypod --port=80 --type=NodePort $do
  • Use kubectl explain pod.spec.containers.securityContext — in-exam docs
  • Practice on killer.sh until you score 80%+ consistently
Study Resources: Killer.sh (must), KodeKloud CKA course, Mumshad Mannambeth (Udemy), play on local kind/minikube cluster daily. 2–3 months prep is enough if you're consistent.

CKS — Certified Kubernetes Security Specialist
CERT+00:00:00
▼
0/0
00:00:00
📋 Exam Overview
  • Duration: 2 hours — hands-on, performance-based (no MCQs)
  • Prerequisite: Active CKA certification — CKS cannot be taken without it
  • Passing score: 67%
  • Cost: $395 USD (includes 1 free retake)
  • Valid: 2 years from date of passing
  • Open-book: kubernetes.io/docs + Falco/AppArmor/seccomp docs allowed
  • Harder than CKA — security mindset required, not just operations
🔒 Cluster Setup & Hardening (15%)
  • CIS Benchmarks for Kubernetes — know the key recommendations
  • kube-bench — automated CIS benchmark checks on nodes
  • Disable anonymous authentication: --anonymous-auth=false on apiserver
  • --authorization-mode=Node,RBAC — never AlwaysAllow
  • Restrict API server access: --insecure-port=0, --bind-address
  • NodeRestriction admission plugin — limit what kubelets can modify
  • Upgrade cluster to fix CVEs — kubeadm upgrade workflow
  • TLS: rotate certificates, --tls-min-version=VersionTLS13
🌐 Cluster Hardening (25%)
  • RBAC deep: least privilege, audit service accounts, remove unused bindings
  • ServiceAccount: automountServiceAccountToken: false when not needed
  • Limit access to etcd — only apiserver should communicate with etcd
  • NetworkPolicy: default deny all, whitelist only required traffic
  • Ingress + Egress NetworkPolicy — know the YAML syntax perfectly
  • GUI tools security: disable dashboard anonymous access, restrict Kubernetes dashboard
🐳 System Hardening (15%)
  • AppArmor: aa-status, load profile, annotate pod with apparmor profile
  • Seccomp: use RuntimeDefault or custom profile in pod securityContext
  • Reduce OS attack surface: disable unused services, remove unnecessary packages
  • Open ports audit: ss -tulpn, netstat — close unused ports
  • Linux capabilities: drop ALL, add only required (e.g., NET_ADMIN)
  • securityContext: privileged: false, allowPrivilegeEscalation: false
🔍 Minimize Microservice Vulnerabilities (20%)
  • Pod Security Admission (PSA): enforce, audit, warn — baseline/restricted/privileged
  • OPA Gatekeeper: ConstraintTemplate + Constraint — policy enforcement
  • Secrets management: never store in env vars as plaintext — use mounted secrets
  • Encrypt etcd at rest: EncryptionConfiguration with aescbc/secretbox provider
  • Container sandbox: gVisor (runsc), Kata Containers — RuntimeClass
  • Multi-tenancy: namespace isolation, LimitRange, ResourceQuota per team
📦 Supply Chain Security (20%)
  • Trivy: trivy image nginx:latest — scan before pushing to registry
  • ImagePolicyWebhook — admission webhook to block vulnerable images
  • Minimize base image: use distroless or alpine — fewer attack vectors
  • Docker trust / image signing — verify image integrity
  • Allowed registries: only pull from approved registry (OPA policy)
  • SBOM (Software Bill of Materials) — trivy sbom generation
🕵️ Runtime Security (20%)
  • Falco: real-time threat detection using syscall rules
  • Falco rules: /etc/falco/falco_rules.yaml — detect shell in container, file write
  • Custom Falco rule: - rule: "Detect Shell" condition: spawned_process and proc.name=bash
  • Falco outputs: syslog, file, HTTP endpoint, Slack webhook
  • Audit logging: --audit-log-path, --audit-policy-file — log API requests
  • Audit policy levels: None, Metadata, Request, RequestResponse
  • Immutable containers: readOnlyRootFilesystem: true in securityContext
  • Detect CVEs in running pods: trivy k8s --report all cluster
⚡ Exam Speed Tips
  • Same alias trick: alias k=kubectl && export do="--dry-run=client -o yaml"
  • Practice on killer.sh — CKS simulator is very close to actual exam
  • Bookmark: AppArmor profiles path (/etc/apparmor.d/), Falco config path
  • Know how to find and edit kube-apiserver static pod: /etc/kubernetes/manifests/kube-apiserver.yaml
  • Time check: skip and flag hard questions, return after finishing easier ones
Study Resources: KodeKloud CKS course (Mumshad), killer.sh CKS simulator, Kim Wüstkamp's CKS notes on GitHub. Give 2–3 months after CKA. Practice Falco, AppArmor, Seccomp hands-on every day — you can't memorize these, you need muscle memory.

📁 File System & Navigation
  • ls, ls -la, cd, pwd, mkdir, rm, cp, mv, touch

  • Hard links vs Symbolic links — ln vs ln -s
  • df -h, du -sh — disk usage
  • lsblk, fdisk, mount, umount — block devices
⚙️ Process Management
  • ps aux — list all processes with details
  • top, htop — interactive process viewer
  • kill PID, kill -9 PID, pkill name, killall
  • jobs, bg, fg — background/foreground jobs
  • nohup command & — run after logout
  • nice, renice — process priority
  • systemctl start/stop/restart/status/enable/disable service
  • journalctl -u servicename -f — service logs
  • crontab -e — schedule jobs (syntax: min hr day mon dow)
  • at command — one-time scheduled tasks
🌐 Networking
  • ip addr show, ip link, ip route — modern net tools
  • ifconfig, route — older tools (still asked)
  • netstat -tulnp — open ports and listeners
  • ss -tulnp — modern replacement for netstat
  • curl -X POST -H "Content-Type: application/json" -d '{}'
  • wget, curl -O — download files
  • ping, traceroute, mtr — connectivity testing
  • nmap -sV IP — port scan and service detection
  • iptables -L, iptables -A INPUT -p tcp --dport 80 -j ACCEPT
  • firewalld: firewall-cmd --add-port=80/tcp --permanent
  • ssh, ssh-keygen, ssh-copy-id, scp, rsync
  • /etc/hosts, /etc/resolv.conf, DNS troubleshooting
📝 Text Processing (Interview Favourite)
  • grep -r "pattern" /path — recursive search
  • grep -v, -i, -n, -E (extended regex)
  • awk '{print $1, $3}' — column extraction
  • awk -F':' '{print $1}' /etc/passwd — custom delimiter
  • sed 's/old/new/g' file — find and replace
  • sed -n '5,10p' file — print specific lines
  • cut, sort, uniq, wc -l, head, tail
  • tail -f /var/log/syslog — live log monitoring
  • xargs, tee, pipe chaining
🔍 Troubleshooting (Must Know)
  • High CPU: top → find PID → strace -p PID
  • High Memory: free -h, vmstat, /proc/meminfo
  • Disk full: df -h, du -sh /var/log/* → find & delete
  • Load average: uptime — 1/5/15 min averages
  • dmesg | tail — kernel messages, hardware errors

📌 Core Commands
  • git init, git clone URL, git remote -v
  • git add ., git commit -m "msg", git push origin branch

🌿 Branching
  • git branch, git checkout -b newbranch
  • git merge branchname — merge with commit
  • git rebase main — reapply commits on top
  • Rebase vs Merge: rebase = linear history, merge = preserves history
  • git branch -d, git branch -D (force delete)
🚀 Advanced Operations
  • git stash, git stash pop, git stash list
  • git cherry-pick commitHash — apply specific commit
  • git reset --soft HEAD~1 — undo commit, keep changes staged
  • git reset --hard HEAD~1 — undo commit, discard changes
  • git revert commitHash — safe undo (creates new commit)
  • git tag v1.0.0, git tag -a v1.0.0 -m "Release"
  • git bisect — find which commit introduced a bug
  • git blame filename — who changed each line
🔄 Branching Strategies
  • GitFlow: main, develop, feature/*, hotfix/*, release/*
  • Trunk-based: everyone commits to main, short-lived branches
  • GitHub Flow: main + feature branches + PR
  • Git Hooks: pre-commit (lint), pre-push (tests), commit-msg
  • PR workflow: fork → branch → PR → review → merge
  • .gitignore — ignore node_modules, .env, *.log
😱 Resolving Conflicts
  • git merge → conflict → edit file → git add → git commit
  • Use VS Code / IntelliJ merge conflict UI
  • git mergetool — graphical conflict resolution
Interview Tip: "Rebase vs Merge?" — Rebase rewrites history for clean linear log (use locally). Merge preserves full history (use for PRs).

4. Kubernetes (CKA + CKS)
MUST — CERTIFIED00:00:00
▼
0/0
00:00:00
🏗️ Architecture
  • Control Plane: API Server, etcd, Scheduler, Controller Manager
  • Worker Node: kubelet, kube-proxy, container runtime
  • etcd: distributed key-value store — stores all cluster state
  • API Server: entry point for all kubectl commands
📦 Core Workloads
  • Pod: smallest unit — one or more containers sharing network/storage
  • ReplicaSet: ensures N replicas running always
  • Deployment: ReplicaSet + rolling updates + rollback
  • StatefulSet: ordered pods with stable identity (DBs)
  • DaemonSet: one pod per node (logging, monitoring agents)
  • Job: run-to-completion tasks; CronJob: scheduled jobs
  • kubectl rollout status/history/undo deployment/myapp
🌐 Networking & Services
  • ClusterIP: internal only — default service type
  • NodePort: exposes service on node IP + static port (30000-32767)
  • LoadBalancer: creates cloud LB (AWS ELB, GCP LB)
  • ExternalName: DNS alias to external service
  • Ingress: L7 HTTP routing — path-based, host-based
  • Ingress Controller: nginx, traefik, AWS ALB controller
  • NetworkPolicy: whitelist pod-to-pod traffic (ingress/egress rules)
  • DNS: service.namespace.svc.cluster.local
⚙️ Config & Secrets
  • ConfigMap: non-sensitive config — env vars or volume mount
  • Secret: base64 encoded — Opaque, TLS, docker-registry
  • Inject as env: envFrom: - configMapRef / secretRef
  • Mount as volume: /etc/config/key = value
  • External secrets: AWS Secrets Manager via ESO
💾 Storage
  • PersistentVolume (PV): actual storage (NFS, EBS, EFS)
  • PersistentVolumeClaim (PVC): pod requests storage
  • StorageClass: dynamic provisioning (gp2, gp3 on AWS)
  • AccessModes: ReadWriteOnce, ReadOnlyMany, ReadWriteMany
📐 Scheduling
  • Resource requests & limits: CPU (500m), Memory (256Mi)
  • LimitRange: default limits per namespace
  • ResourceQuota: max resources per namespace
  • Taints: kubectl taint nodes node1 key=value:NoSchedule
  • Tolerations: pod can be scheduled on tainted node
  • Node Affinity: prefer/require specific nodes
  • Pod Anti-Affinity: spread pods across nodes/zones
📈 Autoscaling
  • HPA: scale pods based on CPU/memory/custom metrics
  • VPA: resize pod resources automatically
  • KEDA: event-driven autoscaling (queue-based)
  • Cluster Autoscaler: add/remove nodes automatically
🔐 RBAC & Security (CKS)
  • Role: permissions in a namespace; ClusterRole: cluster-wide
  • RoleBinding, ClusterRoleBinding — bind role to user/SA
  • ServiceAccount: identity for pods to call K8s API
  • PodSecurityContext: runAsNonRoot, readOnlyRootFilesystem
  • NetworkPolicy: deny all + allow specific (zero trust)
  • Admission Controllers: OPA/Gatekeeper policy enforcement
  • Falco: runtime threat detection in K8s
  • Image scanning before deploy (Trivy in pipeline)
🪖 Helm
  • helm install, upgrade, rollback, uninstall, list
  • helm create mychart — scaffold chart structure
  • values.yaml — default values; override with -f or --set
  • Templates: deployment.yaml, service.yaml with {{ .Values.* }}
  • helm repo add, helm search repo
🐛 Troubleshooting (Most Asked)
  • CrashLoopBackOff: kubectl logs pod → check exit code → fix app
  • OOMKilled: increase memory limit or fix memory leak
  • Pending Pod: kubectl describe pod → check events (resources? taints?)
  • ImagePullBackOff: wrong image name or missing registry secret
  • kubectl describe, logs, exec, top pod, get events
Interview Tip: Always answer troubleshooting Qs with: describe → logs → events → exec into pod → check networking. Show a systematic approach.

⚡ Core Concepts
  • Provider: plugin to talk to AWS/GCP/Azure (aws, azurerm, google)
  • Resource: infrastructure object (aws_instance, aws_s3_bucket)
  • Data Source: read existing infra (data "aws_ami" "latest")
  • Variable: input parameters — var.region, var.instance_type
  • Local: computed local values — locals { name = "myapp" }
  • Output: expose values after apply — output "ip" { value = aws_instance.main.public_ip }

  • EBS gp3 vs gp2 — gp3 is cheaper and faster

  • Managed policies vs inline policies
  • IRSA: IAM Roles for Service Accounts (EKS)

📝 Basics
  • Shebang: #!/bin/bash — must be first line

🔀 Control Flow
  • if [ condition ]; then ... elif ... else ... fi
  • String: [ "$a" == "$b" ], [ -z "$a" ] (empty), [ -n "$a" ] (non-empty)
  • Numbers: [ $a -eq $b ], -ne, -lt, -gt, -le, -ge
  • File: [ -f file ] (exists+file), [ -d dir ] (directory), [ -r file ] (readable)
  • case $var in pattern1) ;; pattern2) ;; *) ;; esac
  • for i in {1..10}; do echo $i; done
  • for file in /var/log/*.log; do ...; done
  • while [ condition ]; do ...; done
  • until [ condition ]; do ...; done
  • break, continue — loop control
🔧 Functions
  • function name() { ... } or name() { ... }
  • Arguments inside function: $1, $2, $@
  • return value via exit code: return 0 (success), return 1 (failure)
  • Return string: echo result from function; capture with $(func)
  • Local variables: local var="value" — prevent global pollution
🚨 Error Handling
  • set -e — exit on first error
  • set -u — exit on undefined variable
  • set -o pipefail — fail if any pipe command fails
  • trap 'echo "Error on line $LINENO"' ERR
  • trap cleanup EXIT — run cleanup on exit
  • command || { echo "failed"; exit 1; }

  • Backup script: tar + timestamp + rsync to S3
  • Log rotation: find /var/log -name "*.log" -mtime +7 -delete

  • Disk alert: df -h | awk '$5>80{print "Disk "$5" on "$6}'

  • String manipulation: ${var#prefix}, ${var%suffix}, ${var/old/new}

  • Read from file: while IFS= read -r line; do ...; done < file
